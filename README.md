# Cryptography Demo App

## Overview

This project is a simple Python-based cryptography demonstration that shows three important security concepts:

1. **SHA-256 hashing**
2. **Substitution cipher encryption/decryption using a Caesar cipher**
3. **Digital signature generation and verification using OpenSSL**

The purpose of this app is to demonstrate how data can be hashed, encrypted in a simple way, and digitally signed to confirm authenticity and integrity.

---

## Objectives

This application was created to satisfy the following requirements:

- Generate SHA-256 hashes for input strings or files
- Encrypt and decrypt text using a simple substitution cipher
- Simulate a digital signature process using OpenSSL
- Show how basic cryptographic tools support security concepts such as integrity, confidentiality, and authenticity

---

## Features

### 1. SHA-256 Hashing
The program can generate a SHA-256 hash for:

- a user-defined string
- a file stored on the computer

SHA-256 is a one-way hashing algorithm. It produces a fixed-length output that uniquely represents the data. If the data changes, the hash changes completely.

### 2. Caesar Cipher Encryption and Decryption
The app includes a Caesar cipher, which is a basic substitution cipher.

- Each letter in the plaintext is shifted by a fixed number of positions
- The encrypted text can later be decrypted by shifting the letters back

This is not secure for real-world use, but it is useful for demonstrating the basic concept of encryption and decryption.

### 3. Digital Signature Simulation with OpenSSL
The program uses OpenSSL to:

- generate an RSA private key
- extract the matching public key
- sign a file using the private key
- verify the signature using the public key

This demonstrates how digital signatures help confirm:

- the file came from the expected sender
- the file was not changed after it was signed

## How the Program Works

### SHA-256 Hashing Process
The program first demonstrates hashing by:

1. Taking a sample input string
2. Computing its SHA-256 hash
3. Displaying the resulting hexadecimal hash

Then it:

1. Creates a sample text file
2. Reads the file in binary mode
3. Computes the SHA-256 hash of the file contents
4. Displays the file hash

This shows that both strings and files can be hashed for integrity checking.

---

### Caesar Cipher Process
The Caesar cipher portion of the program:

1. Defines a plaintext message
2. Uses a shift value such as `3`
3. Encrypts the plaintext by shifting each letter
4. Decrypts the encrypted text by reversing the shift

For example:

- Plaintext: `Attack at Dawn!`
- Shift: `3`
- Encrypted: `Dwwdfn dw Gdzq!`
- Decrypted: `Attack at Dawn!`

This demonstrates the idea of reversible encryption.

---

### Digital Signature Process
The digital signature portion of the program uses OpenSSL commands through Python.

The process is:

1. Generate a 2048-bit RSA private key
2. Generate a corresponding public key
3. Sign the sample file using the private key
4. Verify the signature using the public key

If the file has not been changed, verification succeeds.

If the file is modified after signing, verification fails.

This demonstrates the security concept of **integrity** and also supports **authenticity**.
